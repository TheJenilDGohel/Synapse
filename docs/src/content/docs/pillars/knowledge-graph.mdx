The Knowledge Graph is Synapse's architectural memory. It tracks entities and their relationships over time, allowing agents to understand not just what the code is now, but why it was built that way and how its dependencies have evolved.

<KnowledgeGraphHighlights client:load />

## Temporal Triples
Synapse uses a **subject-predicate-object** model with built-in temporal awareness.
- **`valid_from` / `valid_to`**: Every fact has a lifespan.
- **Bi-temporal Tracking**: We track both when a fact happened in the project and when it was recorded in Synapse.
- **Contradiction Detection**: Automatic flagging when a new fact contradicts an active one.

## Multi-Hop Traversal
Understand the "ripple effect" of changes by traversing the graph:
- **`graph_traverse`**: Discover all entities connected to a starting point within N hops.
- **`graph_bridges`**: Find entities that link different projects or domains together.
- **Root Cause Analysis**: Trace a bug back through its dependency chain in the graph.

## Time-Travel Queries
Use the `as_of` parameter to query the graph at any point in history.
- **`kg_as_of`**: "What were the project dependencies as of last March?"
- **`kg_timeline`**: "Show me every change to the `AuthModule` architecture over the last year."

## Agent Integration
The Knowledge Graph isn't just for human reading. Tools like `agent_prime` use it to:
- Identify relevant stakeholders for a specific module.
- Surface past architectural ADRs (Architecture Decision Records) related to a task.
- Prevent regressions by surfacing "lessons learned" from similar past features.
