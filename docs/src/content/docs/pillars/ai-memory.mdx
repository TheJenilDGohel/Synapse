Synapse provides a durable memory layer that allows AI agents to learn from every interaction. Unlike ephemeral conversation history, Synapse Memory is structured, searchable, and persists indefinitely.

<AiMemoryHighlights client:load />

## Cross-Session Recall
Agents use `memory_recall` to pull relevant "lessons learned" from previous sessions.
- **Task Context**: Automatically rehydrate the agent with information relevant to the current task.
- **Experience Base**: Build a library of successful patterns and common pitfalls.

## Semantic Deduplication
Synapse doesn't just store everything; it **thinks** about what it's storing.
- **Smart Merging**: Similar memories are automatically combined to prevent context bloat.
- **Importance Scoring**: High-signal insights are prioritized over routine logs.

## Nests and Branches
Organize memory for scale using a biological taxonomy:
- **Nests**: Top-level containers (e.g., "Project A", "R&D", "Personal").
- **Branches**: Topic-specific folders within a nest (e.g., "Auth", "Database Migration").
- **Agent Isolation**: Keep one agent's working notes separate from another's.

## Ingestion Pipelines
Bring existing knowledge into the memory layer:
- **`ingest_markdown`**: Convert meeting notes or RFCs into searchable memories.
- **`ingest_json`**: Import conversation exports from other tools.
- **`capture_outcome`**: Explicitly save the "Winner" state at the end of a successful task.
