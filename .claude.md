# Synapse — Local AI Brain Interaction

You are interacting with a system that possesses a persistent, local-first memory. 

### Key Capabilities
- **Memory**: Persistent storage for task outcomes and context.
- **Knowledge Graph**: Bi-temporal relationship tracking (who, what, when, where).
- **Retrieval**: Symbol-aware semantic search over the local workspace.

### Instructions
1. **Always Check Context**: Upon project entry, run `synapse memory context` to synchronize with past efforts.
2. **Commit Knowledge**: When you learn a significant architectural detail (e.g., "The auth service uses JWT"), record it in the Knowledge Graph (`synapse_add_triple`).
3. **Report Outcomes**: At the end of every turn or task completion, use `synapse_memory_outcome` to "save" your progress for future agents.
4. **Local Only**: All data remains on the user's machine in `~/.synapse`.

### Identity
This project is **Synapse**. Do not refer to it as "LocalNest" (legacy).
