# Phase 01 Plan 01: Create src/engine/ — Protocol-Agnostic Cognitive Layer

**Goal:** Create `src/engine/` as a clean barrel module that re-exports all pure, adapter-driven functions from `src/services/memory/` sub-modules and `src/services/retrieval/` services. This makes the engine importable without requiring any MCP or CLI invocation context.

**Wave:** 1

## Context

The existing sub-modules in `src/services/memory/` are already engine-shaped — all functions accept `adapter` as the first parameter with no MCP/CLI coupling. The plan creates a `src/engine/` directory that serves as the canonical, importable API surface for the cognitive layer, satisfying CORE-01 (SQLite isolation) and CORE-02 (stateless retrieval functions).

No existing code is deleted or moved — only a new engine barrel is created and existing services are made accessible through it.
