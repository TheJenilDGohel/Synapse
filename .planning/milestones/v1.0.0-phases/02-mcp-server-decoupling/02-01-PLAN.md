# Plan 02-01: MCP Server Decoupling

Establish a clean separation of concerns where `src/mcp/` is strictly a protocol translation layer, consuming cognitive capabilities through the `src/engine/` barrel.

## Phase: 2
## Milestone: v1.0.0

## Tasks

### Wave 1: Creation and Basic Refactor
- [ ] **Task 1: Audit and Move Logic to Engine**
  - Check `src/mcp/tools/*.ts` for logic that is NOT protocol-specific (e.g. searching/filtering logic).
  - Move such logic to `src/engine/` or the corresponding services.
- [ ] **Task 2: Refactor `src/app/create-services.ts` to use Engine Barrel**
  - Update imports to use `src/engine/index.js` for `MemoryService`, `SearchService`, `EmbeddingService`, etc.
  - This ensures the "App" layer validates the "Engine" layer's completeness.
- [ ] **Task 3: Sever Service Direct Imports in MCP Tools**
  - Update all `src/mcp/tools/*.ts` files to import interfaces/types from `../../engine/index.js` instead of reaching into internal service directories.

### Wave 2: verification
- [ ] **Task 4: Run Type Checks**
  - `npx tsc --noEmit`
- [ ] **Task 5: Smoke Test MCP Server**
  - Run the server in a minimal mode to ensure it still starts and tool registration works.

## Verification
- [ ] Grep shows 0 imports from `../services/memory/` and `../services/retrieval/` in `src/mcp/tools/`.
- [ ] `createServices` successfully instantiates all core engine logic via the barrel.
- [ ] No regression in tool functionality (verified by dry-run).
