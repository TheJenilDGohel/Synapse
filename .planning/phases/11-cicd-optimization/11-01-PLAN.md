    Refactor the `scripts` section to provide granular access to quality checks:

      - `quality:check-scripts`: The node --check logic currently inside `check` script.

    - Add `test:unit` as an alias for `npm test`.

  </action>

<task type="auto">
  <name>Task 2: Refactor CI workflow for parallelism and path-filtering</name>
  <files>.github/workflows/ci.yml</files>
  <action>
    Rewrite `.github/workflows/ci.yml` with the following optimizations:
    1. **Path Filtering**: Add `on.pull_request.paths-ignore` for `['docs/**', '**.md']` to skip CI on docs-only changes.
    2. **Job: setup**: (Optional but recommended) A job to run `npm ci` and cache `node_modules`. Or just use `actions/setup-node` caching in every job.
    3. **Job: quality**: Split into parallel jobs or use a matrix for: `lint`, `typecheck`, `audit`, `circular-deps`, `package-check`.
    4. **Job: unit-tests**: Matrix OS [ubuntu, windows, macos] x Node [18, 20, 22].
    5. **Job: integration-tests**: Matrix OS [ubuntu, windows, macos] x Node [22]. (Installation is OS-specific but usually Node-version independent enough to run on latest).
    6. **Job: e2e-tests**: Matrix OS [ubuntu, windows] x Node [22]. (Reduced matrix to prevent timeouts).
    7. **Caching**: Ensure `actions/setup-node` with `cache: 'npm'` is used in all jobs.
  </action>
  <verify>
    <automated>gh workflow run ci.yml --ref $(git branch --show-current)</automated>
  </verify>
  <done>ci.yml is updated with parallelized jobs and reduced E2E matrix.</done>
</task>
