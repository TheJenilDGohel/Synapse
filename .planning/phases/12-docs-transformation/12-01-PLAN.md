```markdown
# Synapse - Unified Context Layer for AI Agents

Synapse is an MCP server that combines persistent AI memory, a temporal knowledge graph, and semantic code intelligence.

## Core Capabilities
- **Memory**: Long-term recall with semantic deduplication (Nests).
- **Knowledge Graph**: Entity-triple store with temporal traversal.
- **Search**: Hybrid BM25 + vector search across codebases.
- **Symbol Intelligence**: AST-aware tracking of definitions, usages, and callers.

## Installation
npm install -g synapse-cortex
synapse setup
synapse doctor

## Power Controllers (Tools)
- `synapse_memory_manage`: CRUD + ingestion for memories.
- `synapse_memory_query`: Search + context synthesis.
- `synapse_search`: Unified code/file search.
- `synapse_symbol_query`: Precise AST symbol lookups.
- `synapse_workspace_manage`: FS exploration and scoped reads.
- `synapse_system_manage`: Indexing, health, and maintenance.

## Versioning
CalVer: YYYY.MM.Patch (e.g., 2026.05.0)
```

- [ ] **Step 2: Create SKILL.md**
Define the operational rules for AI agents using Synapse.

```markdown
# Synapse Agent Skill

Operational guidance for AI agents interacting with a Synapse-equipped workspace.

## Context Injection Workflow
1. **Prime**: Always call `synapse_agent_prime` at the start of a task.
2. **Recall**: Use `synapse_memory_query` with `item_format: 'compact'` to find relevant history.
3. **Search**: Use `synapse_search` for initial discovery, then `synapse_symbol_query` for precise mapping.

## Rules
- **Respect Nests**: Use the current task or branch name as the `nest` when storing memories.
- **Efficiency**: Prefer `lite` or `compact` formats for large queries.
- **Verification**: Call `synapse_memory_manage` (outcome) after completing a major step.
```
